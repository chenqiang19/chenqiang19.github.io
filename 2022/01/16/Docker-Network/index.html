Docker容器相关的网络学习笔记

参考链接

容器只是隔离和受限的 Linux 进程，运行容器并不需要镜像。相反，要构建映像，我们需要运行一些容器。

重点关注的问题：

  • 如何虚拟化网络资源，让容器认为每个容器都有一个专用的网络堆栈？
  • 如何将容器变成友好的邻居，防止它们干扰，并能较好的沟通？
  • 如何从容器内部到达外部世界（例如互联网），连接外网？
  • 如何从外部访问在机器上运行的容器（又名端口发布）？

单主机容器网络只不过是众所周知的 Linux 设施的简单组合：

  • network namespaces;
  • virtual Ethernet devices (veth);
  • virtual network switches (bridge);
  • IP routing and network address translation (NAT)

一、使用网络命名空间隔离容器

什么构成了Linux网络堆栈？1，网络设备；2，路由规则集(netfilter、iptables)

用于容器隔离的 Linux 命名空间之一称为网络命名空间。man ip-netns中指出，“网络命名空间在逻辑上是网络堆栈的另一个副本，具有自己的路由、防火墙规则和网络设备。”

二、使用虚拟以太网设备 (veth) 将容器连接到主机

如果无法与专用的网络堆栈通信，专用的网络堆栈就不会那么有用了。 幸运的是，Linux 提供了一个合适的工具——虚拟以太网设备。man中提到，veth 设备是虚拟以太网设备。它们可以充当网络命名空间之间的隧道，以创建到另一个命名空间中的物理网络设备的桥接，但也可以用作独立的网络设备。

Linux 有很多方法来填充路由表。 其中之一是从直接连接的网络接口中提取路由。

四、与外界联系（IP 路由和伪装）

我们的容器可以相互通信。 但是它们不可以与主机(root命名空间)通信。root网络命名空间同样如此。为了在根命名空间和容器命名空间之间建立连接，我们需要将 IP 地址分配给桥接网络接口：

一旦我们将 IP 地址分配给网桥接口，我们就会在主机路由表上获得一条路由。容器可能还可以 ping 网桥接口，但它们仍然无法访问主机的 eth0(不能与外界进行通信)。 我们需要为容器添加默认路由。

这种变化基本上把主机变成了路由器，桥接接口变成了容器的默认网关。
